# Post Task Fit Report

原則`reports/working/YYYYMMDD-HHMM-<ascii-summary>.md`へ次を記録する。

1. 受けた指示と対象要件。
2. 採用した判断と非対象。
3. 実施内容と成果物。
4. 実際に実行したcommandと結果。
5. 指示へのfit。未検証があれば満点にしない。
6. 未対応、外部gate、制約、残余risk。

内部推論の逐語記録ではなく、検収可能な判断根拠を記載する。未実行のtestや未取得のapprovalを合格として書かない。
